常见词释义

简称 全称 说明
CTF Capture The Flag 夺旗赛，从题目信息中获取flag并按照指定格式提交确认
CTE Capture the Ether 区块链智能合约，一般是漏洞利用
MISC miscellaneous 杂项，CTF题目常见题目类型之一
RE Reverse 逆向，CTF题目常见题目类型之一
Crypto Cryptography 密码学，CTF题目常见题目类型之一
PWN 发音类似“砰”，是指攻破设备或者系统 系统/硬件破解，CTF题目常见题目类型之一
WEB website 网站破解，CTF题目常见题目类型之一
Mobile - 安卓破解，CTF题目常见题目类型之一
区块链 - 以太坊智能合约，CTF题目常见题目类型之一
Steg/Stego steganography 隐写术，隐藏术
wp writeup 解题思路，题目解题过程的记录
pl playload （有效攻击负载）是包含在你用于一次漏洞利用（exploit）中的ShellCode中的主要功能代码
shellcode - 可提权代码
exp exploit 漏洞利用，一般是个demo程序
poc proof of concept 漏洞证明，一般就是个样本
vul vulnerable 泛指漏洞
cve Common Vulnerabilities & Exposures 公共漏洞和暴露，漏洞字典，国际上的
cnvd - 漏洞编号，漏洞字典，中国的
0day Zero-dayexploit 没打补丁，没公开的安全漏洞
ak all kill 某一类型题目或者全部题目都被解出
tql taiqiangle “太强了”的拼音首字母，常用于复读
ddddhm daidaididihaoma “带带弟弟好吗”的拼音首字母，常用于复读
yyds yongyuandishen “永远滴神”的拼音首字母，常与大佬昵称连用，可用于复读
一血 first blood 通常指比赛中最先得分/解出某题
套娃 - 题目有很多层，解完一层还有一层，类似俄罗斯套娃
容器 某类或某些题目解题需要创建出的在线环境，由于需要消耗服务器资源，一般有时间、并行数、操作频率等限制
动态积分 - 比赛中用来衡量成绩的得分，解出人数越多该题目的分数值就越低
非预期 - 在出题人预期的解题方法之外的可行解法
签到题 - 一般指比赛中送分的题目，可用于统计实际参赛人数或团队数
py - 常用工具类编程语言python。有时也指比赛中通过非解题手段，从其他渠道或选手处违规获得flag或解题提示
出题人 - 题目（问题）的制造者，经常因为题目难度或脑洞过大被做题者口头威胁，是大家喜闻乐见的迫害对象
萌新/萌旧 - 一些很厉害的人自谦的称呼
大佬 - 恭维一些厉害的人的称呼
师傅 - 有问题请教一些不认识的人时的称呼，可用“大佬”替换。或前面加姓/id称呼德高望重的前辈
xxx爷爷 - 公认的对某一类型题目精通的大佬的称呼
菜鸟/菜狗/菜鸡 - 新入门或想入门的人的自称，有时也是一些大佬的自嘲
小白 - 从没接触过ctf出于好奇想入门的人
弟弟 - 在想要得到别人帮助时的自称
妹子/小姐姐 - 一类是比赛的客服。另一类是参赛选手，常被团队用来招新或大佬婉拒带人
菜 - 技术差，与“大佬”对应，是水群的常用语

骚姿势

不同类型题目getflag技巧

0. 通用技巧

根据上下文盲猜flag

  1. XCTF-2020-高校战疫，签到题flag是‘flag{shijiejiayou}’，另一道叫“武汉加油”的题目的flag是flag{zhong_guo_jia_you}
  2. 校内赛/萌新赛，题目一般备注出题人的昵称，这些昵称有时也会出现在flag中，flag不完整时可以试试手动补全
  3. 非动态容器题目的flag一般是“可读的”，可以尝试 微调/补全 字词

某些高校赛有些缺心眼的新手会在比赛中发布wp，比赛快结束时看一下有没有公开的wp，比如某破解论坛

1. Reverse/Pwn

这两类题有时flag以明文存储在附件文件中，直接用文本编辑器打开搜索关键词即可，尤其是做出人数接近签到人数时

2. misc

对于题目的附件中文件很多的题目，比如游戏题，一般情况下可以着重分析修改时间最晚的文件

身体强壮的可以手动解题。例如：游戏类题目手动通关、手动解300层有密码压缩包、人工爆破flag

3. web

对于共享环境的题目可以直接利用其它选手上传的脚本或直接读取输出文件中的flag，甚至对于杂项题使用同一个在线工具，可以爬取前人的使用记录。

web考查知识点是有限的，有的题目有特殊的正则表达式或其他关键词，可以用搜索引擎搜索这些比较有代表性的关键词查找类似题目甚至相同的题目wp。

混排名，混进复赛/决赛技巧

写的有点乱，可以每次比赛前看一遍，对号入座，做到心里有数。

  1. 奖金贼离谱、进线下队伍有数且线下得奖队伍数占70%以上、为响应国家政策举办
    放平心态，不要因为题目脑洞或者比赛环境而影响自己的心情。除非队伍实力十分强劲，否则看个热闹就好，这些奖金不是给你准备的。

  2. 动态积分、比赛时间较短、限制队员人数较少、题目较多
    这类比赛，往往做出一两道高分题（做出人数少）就能晋级/获奖。参赛人数/队伍数越多越有效（简单题分会降到更低），参赛人数特别多时一般比赛开始前会宣传有多少人（队伍）参赛。

  3. 题目较少或题目难度较大、相同分数的较多
    最后一个flag的提交时间对排名影响最大，所以不要屯flag，除非你的分数不在同分较多的范围内。注意紧盯最后调查问卷（签退题）的放出，如果有的话，这很可能是你最后提交的flag

  4. flag提交策略
    屯flag的目的，一是为了可以表演“垂直上分”小装一下，但主要目的是在提升较少排名较为关键时（前几名奖励差别较大、分数徘徊在晋不晋级之间），在比赛快要结束时提交flag打对手一个措手不及。屯flag策略一般在时间较长的比赛中更有效。
    如果前三没有加成，或者前三血有加成，得到flag时前三血已经没了，而且不是上面第三条的情况，这时flag可以屯着，提交时间不影响最终的分数和排名。
    最后提醒一点，如果比赛平台不稳定，需要预留出更多时间提交flag，防止出现比赛结束flag没提交完的尴尬。

学习定位

  1. 如果想长期从事CTF比赛相关的，要么各个方向全面发展，一个人打全场，要么专门学精一两个方向，加入一些优秀的团队。
    ctf发展到现在大多数都是团队赛，一个人就算把一个方向的题目全部ak，总分算下来也很难比得上各方向题目平衡的一般队伍。而且很多优秀的选手都加入了一些排名靠前的队伍，这样只会强者越来越强，弱者越来越弱，每次比赛前几名都是那几个，如果队伍名没出现只能说明他们没参加。虽然这有点“不公平”，但这是大势所趋。

  2. 如果仅是作为业余爱好，选择自己喜欢的方向学习即可

  3. 如果想通过CTF学习、比赛，将来从事相关工作，就要提前考虑工作的内容，选择合适的方向，侧重实际的学习

  4. 友情提示：CTF界也是有骗子的，而且经常是比你还小的骗子，受法律保护的那种，所以保护好自己。
