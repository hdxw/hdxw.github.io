字符编码

ASCII编码

ASCII (American Standard Code for Information Interchange: 美国信息交换标准代码）是基于拉丁字母的一套电脑编码系统，主要用于显示现代英语和其他西欧语言。

0-9: 48-57(0x30-0x39)
A-Z: 65-90(0x41-0x5A)
a-z: 97-122(0x61-0x7A)


栅栏密码

栅栏密码(Rail Fence Cipher)是一种简单的移动字符位置的加密方法，规则简单，容易破解。
栅栏密码的加密方式：把文本按照一定的字数分成n个组，取每组第一个字连起来得到密文1，再取每组第二个字连起来得到密文2…最后把密文1、密文2…连成整段密文。

解密则反推：把密文每n个字分成一组，取每组第一个字连起来得到明文1，再取每组第二个字连起来得到明文2…最后把明文1、明文2…连成整段明文。

通常加密解密时的参数为每组的字数，其中加密时每组字数(a)和解密时每组字数(b)的关系为（分页公式）：(len(plaintext)-1)/a+1=b

凯撒密码

凯撒加密是最简单且最广为人知的加密技术，它是一种替换加密的技术。
待加密的字符串中的字母按照26个字母表的顺序偏移固定的数目，偏移后字母大小写不变，非字母字符不变。
比如a偏移3变成d，Y偏移3变成B

变异凯撒

所有字符按照ASCII表进行偏移，后面的字符偏移量比前面的增加一。

URL编码

只有字母和数字[0-9a-zA-Z]、一些特殊符号“$-_.+!*’(),”、以及某些保留字，才可以不经过编码直接用于URL。

url编码后的格式是%加上hex值（同ascii码的十六进制），当某些特殊符号作为参数在url中时需要进行编码。

中文在url中传输也是需要进行url编码的，但是相同的中文采用不同的编码方式得到的url编码值是不同的。

具体采用哪种编码方式和操作系统、浏览器、网页字符集都有关，但使用Javascript的函数编码的结果是统一的。

Quoted-printable编码

quoted-printable编码方法适用于所传送的数据中只有少量的非ASCII码，例如汉字。
这种编码方法对于可打印的 ASCII 码，除特殊字符等号 “=” 外，都不改变。等号和不可打印的ASCII码以及非ASCII码的数据的编码方法是：将每个字节的二进制代码用两个十六进制数字表示，然后在前面加上一个等号 “=”。这个转hex的结果和utf8编码的结果是一样的，每个汉字转为6位十六进制。而等号 “=” 的二进制代码为 00111101，即十六进制的 3D，因此等号 “=” 的 quoted-printable 编码为 “=3D”。

具体代码实现python中提供了编码解码的库，直接调用即可（编码结果会自动添加换行）

云影密码

01248 密码：该密码又称为云影密码，使用 01248 四个数字。其中 0 用来表示间隔，其他数字相加之和表示字母序号。

如：28=10，124=7，18=9，再用 1->26 表示 A->Z。

仿射密码

仿射密码是一种单表替换密码。

它的加密函数是e(x)=ax+b(mod m)，其中a和m互质（否则可能无法解密），m是字母的数目。

解码函数是d(x)=a-1(x-b)(mod m)，其中a-1是关于模m乘法逆元。

解码也可以正向计算出映射表，然后逆向比对。

曼彻斯特编码

曼彻斯特编码（Manchester Encoding），也叫做相位编码（ Phase Encode，简写PE），是一个同步时钟编码技术，被物理层使用来编码一个同步位流的时钟和数据。

它在以太网媒介系统中的应用属于数据通信中的两种位同步方法里的自同步法（另一种是外同步法），即接收方利用包含有同步信号的特殊编码从信号自身提取同步信号来锁定自己的时钟脉冲频率，达到同步目的。

未解码报文只包含4种数字：569A，实际上就是0b0101 0b0110 0b1001 0b1010，就是曼彻斯特编码中，0和1的两种位的翻转。

曼彻斯特码：从高到低跳变表示“1”，从低到高跳变表示“0”。

差分曼彻斯特码：有跳变为“1”，无跳变为“0”。

当铺密码

商业隐语

建国前商店出售货物，均无明码标价和固定价格，各商号都有自己的隐语暗号进行交易。这种隐语多为十个汉字组成，依次代表l至10十个数码。

商业隐语多以吉利的字眼，如

  • 成山卫顺元商号：招财童子至，利市仙官来
  • 鸿吉栈：鸿吉生财道，同心利如江
  • 俚岛东升德商号：同心在此地，尽力生意兴

也有实用主义的即不管用词是否吉利，但求隐语能变通的，如

  • 石岛复昌和商号：野外云飘笑，记功何鲜效

每个字拆开，左边变成“黑夕雨票竹，言工人鱼交”;而右边则变成 “予卜田风天，己力可羊文”，一套隐语变成三套。

不过，内地文化水平较低，用上述这些问，学徒不易掌握，所以滕家德生祥的暗语，就用字出露的笔画区分。他们的隐语是：“由中人工大，天主井羊非”。不识字不要紧，能数字端出露的笔画即可。

牲口经纪的隐语，除摸指言价外，一至十分别为丁、谭、品、吊、拐、闹、柴、别、弯、卡。但经纪人交易的价格数额都大，一至十的数码是不够用的，所以十一至二十则分别为六厂、六此、六品、六胡、六满·、六树、六柴、六我、六欠、谭子。一百一十至二百，亦用上述代号，不同的是在腔调上十位数拖前不拖后，百位数拖后不拖前。二百以上，又有其他隐语代替。

挑脚、抬轿子的人，则用另外一套汉字来代表1-10的数字：“挖竺春罗悟 交化翻旭田”，字中有字，不难破解。

米行则又有另外一种数字密码：“旦底 断工 眠川 横目 缺丑 断大 皂底 分头 丸空 田心”。

五金行业最有特色，是用颜色来代替数字，与我们现在所用的“加色法”中的颜色编码如出一辙：“棕红橙黄绿 蓝紫灰白黑”。

而在马戏、杂技、武术和镖行中，则用的是：“柳月汪在中 神形张艾菊”。

汉字笔画数

将汉字和数字关联起来，将汉字定义为明文，将数字定义为密文，加密是将数字映射到对应笔画的汉字，解密是将汉字按照笔画映射回数字。

棋盘密码

经典棋盘密码(Polybius)

棋盘密码，又叫波利比奥斯棋盘（Polybius Checkerboard）、Polybius方表。波利比奥斯棋盘是公元前2世纪希腊人Polybius发明的，是最早的，也是最经典的棋盘密码。

原理是把26个字母放合5x5的方格里，其中i、j放在同一个格中。（根据上下文判断 i 和 j）

ADFGX/ADFGVX密码

ADFGX密码使用密文当中的字母命名。之所以选择这五个字母，是因为它们译成摩斯密码时不容易混淆，可以降低传输错误的几率。

ADFGVX密码是德军在第一次世界大战中使用的栏块密码，是ADFGX密码的增补版。ADFGVX密码扩充为6*6的方格，字元有26个英文字母（不再将I和J视为同一个字）以及数字0到9，这个增改是因为以原来的加密法发送含有大量数字的简短讯息有问题。

关于把两个字母看作同一个字的原因和语言有关，德文中将i和j看做一个字母，法文消去的是k或者w，意文因为只有21个字母，英文则一般会省略去z。

敲击码

敲击码(Tap code)是一种以非常简单的方式对文本信息进行编码的方法。因该编码对信息通过使用一系列的点击声音来编码而命名，敲击码是基于5×5方格波利比奥斯方阵来实现的，不同点是是用K字母被整合到C中。

普莱菲尔密码(Playfair)

Playfair Cipher不同的音译还可以叫做波雷费密码，最早的纪录出现在一份1854年3月26日由查尔斯·惠斯登签署的文件。惠斯登的朋友波雷费勋爵普及了这个加密法。普莱菲尔密码(Playfair Cipher)是第一种用于实际的双字替换密码，用双字加密取代了简单代换密码的单字加密。

普莱菲尔密码又称为单方密码(Single Cipher)之后又出现它的升级版Double Playfair，也就是 二方密码 (Two-square Cipher),在之后又有四方密码(Four-square Cipher)

网传的算法在一些细节处理上略有不同，这里采用了比较通用的算法。
如“编制密码表”不同的处理方法有：

  1. 不包含字母Z，而将I和J单独分开。应该是非英语加密，否则上面的例子就没法加密
  2. 处理密钥字符串时如含有Z或者含有Q时去除。上面示例中没有做任何去除，因为没什么必要性。

在“整理明文”时还有一种做法是，若同组的两字母相同，则在中间添加X，然后重新分组，直到同组没有相同的字母。但这种操作对密码强度没有任何作用，还增加了加密的成本，所以上面选择了在两个字母后面都加X的简单做法。

反切码

戚继光声韵加密法，也就是中国古代的反切码。反切码是在古代注音方法“反切法”的基础上创造的。“反切”在汉代出现，它的规则是用两个汉字拼写给一个汉字注音，取第一个字的字母和第二个字的韵母和声调。如“风”，房声切，取“房”的声母“f”和“声”的韵母“eng”，切出“风”这个字的读音是“fēng”。
